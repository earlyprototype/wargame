# @filing – Research and Reference Index

Authoritative store for research, briefs, prompts, benchmarks, and references. Originals live here; other folders may link but should not duplicate or delete these.

## What lives here
- Style bible: `cli_style_bible.md`
- Prompt kit: `cli_lucasarts_prompts.md`
- Main research brief: `cli_lucasarts_research.md`
- Benchmark plan: `cli_benchmark_plan.md`
- Research log (template): `cli_research_log.md`
- Precedents & influences: `precedents_and_influences.md`
- Anchor Zero concept: `anchor_zero_concept.md`
- Transcript/source refs: `transcript_sources.md`

## Conventions
- Keep originals intact; append with dated sections rather than overwriting.
- Prefer short, linkable documents over mega‑docs; cross‑link with relative paths.
- Use clear filenames: `topic_area.md` (e.g. `cli_style_bible.md`).
- Add references/URLs inline; include a short why‑it‑matters note.

## Adding a new note
1) Create a new `.md` in `@filing/` with a descriptive name.
2) Add a short context block at the top: purpose, scope, sources.
3) Link it from this README under “What lives here”.
